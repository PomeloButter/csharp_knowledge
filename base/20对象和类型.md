## 类和结构
- 类和结构实际上都是创建对象的模板，每个对象都包含数据，并提供了处理和访问数据的方法。
- 结构和类的不同点：
    1. 结构是值类型，存储在栈(stack)上，类是引用类型,存储在堆(heap)上。
    2. 结构不支持继承。

- 结构的特点：
    1. 较小的数据类型使用结构可提高性能。
    2. 结构也使用 <code>new</code> 关键字来声明实例。

## 类
- 类包含成员，成员可以是静态或实例成员。
    - 静态成员属于类
    - 实例成员属于对象

- 类的部分成员解析：
    - 常量
        - 常量与类相关（尽管他们没有 <code>static</code> 修饰符），编译器使用真实值代替常量。
    - 索引器
        - 索引器允许对象用访问数字的方式访问。参见第八章

- 字段
    - 最好不把字段声明为Public，最好把字段声明为 <code>private</code> 使用属性来访问字段。

- 属性
    - 属性的概念是：他是一个方法或一对方法，在客户端代码看来，他们是一个字段。
    ```
    class Student{
        private string _name;
        public string Name{
            get { return _name; }
            set { _name = value; }
        }
    }
    ```
    - Name属性包含 <code>get</code> 和 <code>set</code> 访问器，来检索和设置支持字段的值。
    - <code>get</code> 访问器不带任何参数，且必须返回属性声明的类型。也不应为 <code>set</code> 访问器指定任何显示参数，但编译器假定他带一个参数，其类型也与属性相同，并表示为 <code>value</code> 。

    - 下面的实例使用另一种命名约定，字段使用 <code>camel</code> 命名法（首单词全小写，其余头字母大写），属性使用 <code>pascal</code> 命名法（所有单词头字母大写）.
    ```
    private int age;
    public int Age{
        get { return age; }
        set { age = value; }
    }
    ```
    1. 自动实现的属性
        - 如果属性的set 和 get访问器中 `都` 没有任何逻辑，就可以使用自动实现的属性。这种属性会自动实现成员变量，不需要声明字段。
        ```
        public int Age { get; set; }
        ```
        - 自动实现的属性不能在属性设置中验证属性的有效性
        - 自动实现的属性可以使用属性初始化器来初始化
        ```
        public int Age { get; set; } = 42;
        ```
        
        - 注意：get和set任意一项有逻辑都必须使用字段+属性。
        ```
        public int Age{
            get;
            set {
                if(value > 10){
                    //do something
                }
            }
        }//报错
        //correct
        private int _age;
        public int Age{
            get { return _age; }
            set {
                if(value > 10){

                }
                _age = value;
            }
        }
        ```
    
    2. 属性的访问修饰符
        - 属性的访问器可以具有访问修饰符。get和set必须有一个具备属性的访问级别。

    3. 方法
    - 方法的重载：参数的数量或类型不一样
    - 命名的参数：
        - 调用方法时，变量名不需要添加到调用中，但是可以使用命名的参数调用，明确参数含义。
        ```
        public void MoveAndResize(int x, int y, int width, int height);
        r.MoveAndResize(y: 30, x: 40, width:20, height: 40);
        ```
        - 命名的参数调用可以改变参数的位置。
    - 可选参数
        - C#允许给参数提供默认值，提供了默认值的参数是可选参数，在调用时可以不填写。
        ```
        public void TestMethod(int num1 = 0, int num2);
        c.TestMethod(1);//
        ```

    - 个数可变的参数
        - 使用可选参数可以定义数量可变的参数。然而，还有另一种语法允许传递数量可变的参数——这个语法没有版本控制的问题。
        - 声明数组类型的参数，添加 <code>params</code> 关键字，就可以使用任意数量的int参数调用该方法。
        ```
        public void AnyNumberOfArguments(params int[] data);
        c.AnyNumberOfArguments(1,2,3,4,5);
        c.AnyNumberOfArguments(1,2);
        ```
        - 如果 <code>params</code> 关键字与方法签名定义的多个参数一起使用，则 <code>params</code> 只能使用一次，而且它必须是最后一个参数.
        ```
        WriteLine(string format, params object[] arg);
        ```

- 构造函数
    - 一般情况下，如果没有提供任何构造函数，编译器会在后台生成一个默认的构造函数。它能把所有的成员字段初始化为标准的默认值（引用类型为空引用，数值类型为0，bool类型为false)。
    - 如果自己编写了带参数的构造函数，编译器就不会自动提供默认的构造函数，如果需要请自己编写。

    1. 从构造函数中调用其他构造函数
        - C#使用构造函数初始化器实现此功能。语法为在构造函数的参数列表后加上 <code>:this()</code> 
        ```
        public Car(string name,int speed);
        public Car(string name):this(name,100){
            //to do
        }
        ```
        - 注意:构造函数初始化器在构造函数的函数体之前执行。
        - C#构造函数初始化器可以包含对同一个类的另一个构造函数的调用，也可以包含对直接基类的构造函数的调用（用 <code>base</code> 代替 <code>this</code> )。初始化器重不能有多个调用。

    2. 静态构造函数
        - 静态构造函数会在加载类时，由.NET运行库调用它。
        - 类的一些静态字段和属性可以在该构造函数中初始化。
        - 静态构造函数只调用一次。
        - 静态构造函数不需要访问修饰符，不能带任何参数，只能有一个。

- 只读成员
    - 如果不希望在初始化后修改数据成员，就可以使用 <code>readonly</code> 关键字。

    1. 只读字段
        - 带有 <code>readonly</code> 修饰符的字段只能在构造函数中分配值。(还可以在声明时分配值)
        ```
        public class TestClass
        {
            readonly int num = 32;
            readonly static int num2 = 40;
            public TestClass()
            {
                num = 44;
            }
        }
        ```
        - 也就是只读字段有两次定义值的机会，一次是声明时，声明后可以在构造函数中修改一次。
        - 与 <code>const</code> 修饰符不同。编译器通过 <code>const</code> 修饰符，用其值取代了使用它的变量。编译器直到常量的值。只读字段在运行期间通过构造函数指定。
        - 只读字段可以是实例成员，也可以是类成员。
    
    2. 只读属性
        - 在属性定义中省略 <code>set</code> 访问器，就可以创建只读属性。
        ```
        readonly string _name;
        public string Name{
            get => _name;
        }
        ```

    3. 自动实现的只读属性
    -   以下简单的语法可以创建自动实现的只读属性。   
        ```
        public string Id{ get; } = "123";
        ```
    - 还可以通过构造函数来创建
    ```
        public class TestClass
        {
            public string Name { get; }
            public TestClass()
            {
                Name = "test";
            }
        }
    ```

    4. 不可变的类型
        - 如果类型包含可以改变的成员，他就是一个可变的类型。使用 <code>readonly</code> 修饰符，编译器只会在状态改变时报错。状态只能在构造函数中初始化。如果对象没有任何可以改变的成员，只有只读成员，他就是一个不可变类型。这对于多线程是非常有用的。
        - 不可变类型的一个例子是String类。


## 匿名类型
- 匿名类型通过 <code>var</code> 和 <code>new</code> 关键字创建。
```
var anonymous = new {
    name = "jame",
    age = 24
};
```
- 如果创建两个所有属性相同的匿名类型，他们的类型就相同，可以使用=互相设置。
```
var anonymous = new {
    name = "jame",
    age = 24
};
var anonymous2 = new {
    name = "john",
    age = 20
};
anonymous = anonymous2;
```
- 如果所设置的值来自另一个对象，就可以简化初始化器，不必写属性名。他会把另一个对象的属性名自动投影到该匿名类中。
```
var anonymous = new {
    person.Name,
    person.Age
};
anonymous.Age = 24;
```

## 结构

- 有时候我们仅需要一个小的数据结构，这时候我们使用结构可以优化性能。
```
public struct Dimensions{
    public double Length { get; set; }
    public double Width { get; set; }
}
```
- 结构是值类型，不是引用类型。他们存储在 <code>栈</code> 中或存储为 <code>内联</code> （如果他们是存储在堆中的另一个对象的一部分）,其生存期的限制和简单的数据类型一样。
- 结构不支持继承。
- 结构的默认构造函数会自动把成员初始化为其默认值 <code>default(T)</code> ;
- 结构体总有一个隐式的无参构造函数，即使你提供了其他带参构造函数。
- 在C#6中，你可以定义无参构造函数，早期版本则不可。
- 使用结构，可以指定字段如何在内存中布局。

1. 结构是值类型
    - 虽然结构是值类型，但在语法上可以当作类来处理。
    ```
    Dimensions point = new Dimensions();
    point.Length = 24;
    ```
    - 注意，因为结构体是值类型，所以 <code>new</code> 运算符与类和其他引用类型的工作方式不同。 <code>new</code> 运算符并不分配堆中的内存，而是只调用相应的构造函数,根据传递给他的参数，初始化所有的字段。
    - 因此，可以不使用new来初始化结构体。//这里我使用vs2017和vscode都提示使用了未赋值的局部变量“point”，因此存疑
    ```
    Dimensions point;
    point.Length = 24;
    point.Width = 6;
    ```
    

2. 结构和继承
- 结构不是为继承设计的，这意味着结构不能继承。唯一例外是对应的结构最终派生于类System.Object。因此，结构也可以访问System.Object的方法。
- 在结构中，甚至可以重写System.Object中的方法，如重写ToString()方法。
- 结构的继承链是：每个结构派生自System.ValueType类，System.ValueType类又派生自System.Object。ValueType并没有给Object添加任何新成员，但提供了一些更适合结构的实现方式。

3. 结构的构造函数
- 定义方式和类相同。
- 另外，可以像类那样为结构提供Close()或Dispose()方法。

## 按值和按引用传递参数
- 结构按值传递，类按引用传递。
- 使用 <code>ref</code> 或 <code>out</code> 关键字来让值类型引用传递。
```
int a = 0;
Change(ref a);
// a = 2;
Change2(out a);
// a = 3;
public void Change(ref a){
    a = 2;
}
public void Change2(out a){
    a = 3;
}
```
- 使用<code>ref</code> 关键字的变量必须先初始化。
- 使用 <code>out</code> 关键字的变量则不用先初始化。
- ref是用于引用传递
- out则是一般用于返回多个值。
- 返回多个值有多个选项。
    1. 使用类或结构。
    2. 使用元祖类型。
    3. 使用out关键字。

## 可空类型
- 引用类型的变量可以为空，而值类型的变量不能。
- 处理这个冲突的一个方法是把基本类型转换成类（java的方法）。使用引用类型，映射到允许空值的数据库数字，有一个重要的缺点:带来了额外的开销。
    - 对于引用类型： 需要垃圾收集器进行清理
    - 值类型不需要用垃圾收集器清理，变量超出作用域时，从内存中删除
- C#的解决方案:可空类型。可空类型是可以为空的值类型。可空类型只需要在类型后面添加 <code>?</code> 。与基本结构相比，值类型的唯一开销是一个可以确定他是否为空的布尔值成员。
```
int x1 = 1;
int? x2 = null;
x2 = x1;
x1 = (int)x2;
x1 = x2.HasValue?x2.Value:-1;
//还可以使用合并操作符
x1 = x2??-1;
```        

## 枚举
- 枚举是值类型。
- 默认情况下，enum的类型是 <code>int</code> . 这个基本类型可以改为其他整数类型（byte, short, int, 带符号的long和无符号变量)，命名常量的值从0开始底层，但他们可以改为其他值：
```
public enum Color : short
{
    Red = 1,
    Green = 2,
    Blue = 3
}
```
- 使用强制转换可以把数字改为枚举值，把枚举值改为数字。

- 还可以使用enum类型把多个选项分配给一个变量，而不仅仅是一个枚举常量。为此，分配给常量的值必须是不同的位，FLags属性需要用枚举设置。
```
[Flags]
public enum DaysOfWeek
{
    Monday = 0x01,
    Tuesday = 0x02,
    Wednesday =0x04,
    Thursday = 0x08,
    Friday = 0x10,
    Saturday = 0x20,
    Sunday = 0x40
}
```
```
DaysOfWeek day = (DaysOfWeek)3;
//day就是TuesDay
```
- 有了这个枚举类型，就可以使用逻辑或运算符位一个变量指定多个值。
```
DaysOfWeek day = DaysOfWeek.Monday | DaysOfWeek.Tuesday;
WriteLine(day);
//输出Monday,Tuesday
```
- 设置不同的位，也可以结合单个位来包括多个值。如Weekend的值0x60是用逻辑或运算符结合了Saturday和Sunday,Workday则结合了从Monday到Friday的所有日志，AllWeek则是Workday+Weekend.
```
[Flags]
public enum DaysOfWeek
{
    ...
    Weekend = Saturday | Sunday,
    Workday = 0x1f,
    AllWeek = Workday | Weekend

}
```

- 使用枚举，类Enum有时非常有助于动态获得枚举类型的信息。枚举提供了方法来解析字符串获得相应的枚举常数（前面有提过），获得枚举类型的所有名称和值。
```
    Color red;
    Enum.TryPase<Color>("Red",out red);
```
- <code>Enum.GetNames</code> 返回一个包含所有枚举名的字符串数组。
- <code>Enum.GetValues</code> 返回一个枚举值的一个数组。

## 部分类
- <code>partial</code> 关键字允许把类、结构、方法或接口放在多个文件。这在工具自动生成的类中非常有用。如EntityFramework自动生成的实体类，你希望给他加上方法，如果直接写在里面每次更新会自动替换掉，使用 <code>partial</code> 则可以写在其他地方，保证不会被替换。
```
partial class SampleClass{
    public void MethodOne();
}
partial class SampleClass{
    public void MethodTwo();
}
```

- 如果声明类时使用了以下关键字，则这些关键字就必须应用于同一个类的所有部分
    - 访问修饰符 public private protected
    - internal
    - abstract
    - sealed
    - new 
    - 一般约束
- 在嵌套的类型中，只要partial 关键字位于class关键字的前面，就可以嵌套部分类。再把部分类编译到类型中时
    - 属性、XML注释、接口、泛型类型的参数属性和成员会合并。
```
[CustomAttribute]
public partial class Test:BaseClass,IClass
{
    public void MethodOne();
}
[AnotherAttribute]
partial class Test:IOtherClass
{
    public void MethodTwo();
}
=
[AnotherAttribute]
[CustomAttribute]
public class Test:BaseClass,IClass,IOtherClass
{
    public void MethodOne();

    public void MethodTwo();
}
```
- 部分类可以包含部分方法。如果生成的代码应该调用可能不存在的方法，这就是非常有用的。
    - 你可以决定自定义的实现代码
    - 也可以什么都不做
- 如果你没有实现部分方法，编译器将删除这个方法调用：
```
partial class SampleClass{
    public void MethodOne(){
        MethodTwo();
    }
    public partical void MethodTwo();
}
partial class SampleClass{
    public  void MethodTwo(){
        //to do
    }
}
```
- 部分方法必须是 <code>void</code> 类型，否则编译器在没有实现代码的情况下无法删除调用。

## 扩展方法
- 有许多扩展类的方法。继续就是给对象添加功能的好方法。扩展方法是给对象添加功能的另一个选项，在不能使用继承时，也可以使用这个选项。（例如类是密封的）。
- 注意:扩展方法也可以用于扩展接口。这样，实现该接口的所有类就有了公共方法。
- 扩展方法是 **静态方法** ，它是类的一部分，但实际上没有放在类的源码里。
- 假设希望用一个方法扩展string类型，该方法计算字符串中的单词数。
```
public static class StringExtension
{
    public static int GetWordCount(this string s) => s.Split().Length;
}

string fox = "my name is john";
fox.GetWordCount();
```
- 使用 <code>this</code> 关键字和第一个参数来扩展字符串。这个关键字定义了要扩展的类型。
- 即使扩展方法是静态的，也要使用标准的实例方法语法。注意，这里使用了 `fox` 变量而没有使用类型名来调用 `GetWordCount()` 。
- 在后台，编译器会把它改为调用静态方法：
```
int wordCount = StringExtension.GetWordCount(fox);
```
- 编译通过定义扩展方法的静态类所在的名称空间来找到。因此要使用扩展方法必须引用该名称空间。

## Object类
- Equals()(两个版本)和RefreenceEquals()方法：注意有 `3` 个用于比较对象相等性的不同方法，这说明.NET Framework在比较相等性方面有相当复杂的模式。这3个方法和比较运算符 `==` 在使用方式上有微妙的区别。而且，在重写带一个参数的虚Euqals()方法时也有一些限制，因为System.Collections名称空间的一些基类要调用该方法，并希望它以特定的方式执行。
- Finalize()方法：第五章将介绍这个方法，它是最接近C++风格的析构函数，在引用对象作为垃圾被回收以清理资源时调用它。
- MemberwiseClone()方法：这是System.Object中唯一没有在本书其他地方详细论述的方法。不需要讨论这个方法，因为他在概念上相当简单，他只复制对象，并返回对副本的一个引用（对于值类型，就是一个装箱的引用）。注意，得到的副本时一个浅表复制，即它复制了类中的所有值类型。如果类包含内嵌的引用，就只赋值引用，而不复制引用的对象。这个方法是受保护的，所以不能用于复制外部的对象。该方法不是虚方法，无法重写。
